WEEK 2 - DAY 1: ADVANCED PROMPTING
====================================

1. Chain-of-Thought (CoT) Prompting
   - Definition: Asking the model to reason step by step before answering
   - Why it works: Makes reasoning explicit, improves accuracy on complex tasks
   - When to use: Math, logic, multi-step reasoning problems
   - Example: "Let me think step by step: 1. ... 2. ... 3. ... Therefore..."

2. Tree-of-Thought (ToT) Prompting
   - Definition: Exploring multiple reasoning paths before converging on an answer
   - Why it works: Considers alternatives, finds better solutions
   - When to use: Open-ended problems, creative tasks, planning
   - Example: "Approach 1: ... Approach 2: ... Approach 3: ... Best solution: ..."

3. Self-Consistency
   - Definition: Generate multiple answers and take the most consistent one
   - Why it works: Majority vote reduces errors
   - When to use: High-stakes reasoning, ambiguous problems
   - Example: Run 3 times, pick answer that appears most often

4. Meta-Prompting
   - Definition: Asking the model to evaluate and improve its own response
   - Why it works: Self-correction leads to higher quality
   - When to use: High-quality outputs needed, refinement tasks
   - Example: "Evaluate your answer... What would you improve? Improved version:"

5. Prompt Chaining
   - Definition: Breaking a complex task into multiple sequential prompts
   - Why it works: Each step focuses on one subtask
   - When to use: Complex multi-step tasks, document processing
   - Example: Step 1: Extract key points -> Step 2: Expand -> Step 3: Synthesize

6. Negative Prompting
   - Definition: Specifying what NOT to do in the prompt
   - Why it works: Prevents unwanted content, enforces constraints
   - When to use: Constrained generation, avoiding jargon, tone control
   - Example: "DO NOT use: jargon, DO NOT: mention companies"

CO TASK COMPARISON RESULTS:
---------------------------
Without CoT: Faster but error-prone
With CoT: More accurate on reasoning tasks

Key Insight: CoT adds 2-3 seconds but improves accuracy significantly.

WEEK 2 - DAY 2: STRUCTURED OUTPUTS
====================================

Key Concepts Learned:

1. Pydantic v2 Models
   - BaseModel for defining data schemas
   - Field() for field descriptions and validation
   - Type hints for type safety
   - Automatic JSON schema generation

2. Validators
   - @validator: Field-level validation
   - @field_validator: New in v2
   - @model_validator: Cross-field validation
   - Custom validation logic

3. Model Config
   - json_schema_extra: Examples in JSON schema
   - Extra fields handling
   - Field aliases

4. Field Types
   - Literal: Restricted values
   - Optional: Nullable fields
   - List, Dict: Collections
   - EmailStr, HttpUrl: Special types
   - datetime, date: Time types

5. Structured Output Extraction
   - PydanticOutputParser: Parse LLM output to Pydantic
   - get_format_instructions(): Generate JSON schema prompt
   - with_structured_output(): LangChain method

6. Comparison of Methods
   - String Output: Unreliable, no validation
   - JSON Output: Semi-reliable, manual validation
   - Pydantic Output: Reliable, built-in validation
