WEEK 2 - DAY 4: EXTERNAL APIS AS TOOLS
========================================

Key Concepts Learned:

1. HTTP Tools with httpx
   - Async HTTP client
   - GET/POST requests
   - JSON response handling
   - Timeout configuration

2. Error Handling
   - Custom exceptions (APIError, RateLimitError, TimeoutError)
   - Graceful error recovery
   - User-friendly error messages

3. Exponential Backoff
   - Formula: delay = min(base * 2^attempt, max_delay)
   - Jitter: random variation to avoid thundering herd
   - Used for rate limit handling
   - Prevents overwhelming APIs

4. Retry Logic
   - Multiple attempts with increasing delays
   - Different retry strategies for different error types
   - Fallback chains

5. Fallback Chains
   - Primary tool -> Secondary tool -> Tertiary tool
   - Graceful degradation
   - Ensures functionality even when APIs fail

6. Real APIs Integrated:
   - Open-Meteo Weather API (free, no key required)
   - CurrentsAPI News API (free tier)
   - ExchangeRate-API (free tier)

7. News Research Agent
   - Extracts entities (people, organizations, locations, dates)
   - Classifies topics
   - Summarizes findings
   - Multi-step analysis pipeline

8. Tool Execution Logs
   - Track all attempts
   - Record success/failure
   - Debugging and monitoring

APIs Used:
- Weather: open-meteo.com
- News: currentsapi.services
- Currency: exchangerate-api.com

Error Handling Strategies:
1. Retry with backoff
2. Fallback to alternative tool
3. Mock data on complete failure
4. Clear error messages to user
